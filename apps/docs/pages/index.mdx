## Principles

- No middleware! I've always disliked the middleware pattern, for two reasons. The first is that it's difficult to make guarantees about where and the order that middleware run, and secondly it's hard to provide robust type information for middleware. **Instead**, Kaito offers a `.through()` method that allows you to perform logic and swap out context before the request is handled.
